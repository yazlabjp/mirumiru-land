## 1. 概要とねらい

ふわふわと画面内を動きまわる風船を指で追いかけてタッチし、割る動体視力(動く対象を捕捉する目の動き)のアプリ。速さ・大きさ・軌道(まっすぐ/ふらふら)を設定で調整でき、★2のゆっくり大きい設定から★4の速く小さい設定まで難易度が動く。

## 2. 設定スキーマ(完全版・このまま実装する)

```javascript
settings: {
  size: { type: "choice", label: "おおきさ", default: "l",
    options: [ {value:"xl",label:"特大"}, {value:"l",label:"大"},
               {value:"m",label:"中"}, {value:"s",label:"小"} ] },
  speed: { type: "choice", label: "はやさ", default: "normal",
    options: [ {value:"slow",label:"ゆっくり"}, {value:"normal",label:"ふつう"},
               {value:"fast",label:"はやい"} ] },
  motion: { type: "choice", label: "うごきかた", default: "straight",
    options: [ {value:"straight",label:"まっすぐ"}, {value:"wander",label:"ふらふら"} ] },
  count: { type: "range", label: "どうじに出る かず", min: 1, max: 3, step: 1, default: 1 },
  duration: { type: "choice", label: "時間", default: "60s",
    options: [ {value:"30s",label:"30秒"}, {value:"60s",label:"1分"},
               {value:"90s",label:"1分30秒"} ] }
}
```

**数値対応表(規範)**:

| 設定 | value → 実値 |
|---|---|
| size | xl→180px / l→140px / m→110px / s→80px(風船の直径。`VT.Stim.shape`の`size`) |
| speed | slow→80px/秒 / normal→150px/秒 / fast→250px/秒(`VT.Motion.move`の`speed`) |
| motion | straight→風船ごとに1回`VT.Rand.pick(["lineH","lineV"])`で固定(まっすぐ往復) / wander→`"random"`(ふらふらとした軌道) |

- digest対象は size/speed/motion/count/duration(定義順)。例: `"l/normal/straight/1/60s"`。difficulty:falseの設定はなし(絵柄固定のためtheme相当の設定を持たない)。
- 共通設定(sound/reduceMotion/mirror/contrast)はcoreが自動付与する(vt-api §2-2)。mirrorは本アプリの動きに左右対称の意味がないため無効果。reduceMotion:trueでも風船の移動自体(課題に必須の動き)は影響を受けず、ポップ演出のみ簡略化される(vt-api §6)。

## 3. 記録スキーマ

```javascript
record: {
  primary: { key: "hits",    label: "われた かず", betterIs: "higher" },
  extras:  [ { key: "hitRate", label: "命中率", unit: "%" } ]
}
```

- hitRate = ヒット数 ÷(ヒット数 + 空振りタッチ数)× 100(四捨五入で整数%)。タッチが一度もない場合は0。
- 空振りタッチはcommon規則により視覚・聴覚とも一切の反応を出さない(§4参照)。命中率算出のためだけに内部カウントする観察用の値。

## 4. 画面と挙動(共通3画面フローとの差分のみ)

### 4-1. プレイ画面

**開始(onStart)**: `ctx.data`に`{ misses: 0 }`を用意する。`stageTouch: true`とし(空振りタッチ検出のため)、`count`個の風船を下記の出現規則で同時に生成する。

**出現規則(規範)**:
1. `VT.Stim.shape({shape:"circle", color:..., size})`で風船を1個生成し、`VT.Rand.pos({size, area:"full"})`で得た位置に配置する。
2. `motion`設定に応じたpathを決める(straight→`VT.Rand.pick(["lineH","lineV"])`をこの風船について1回だけ選ぶ/wander→`"random"`)。`VT.Motion.move(el, {path, speed, area:"full"})`を呼び、戻り値のctrlを`ctx.addTarget(el, {ctrl})`のdataとして保持する。
3. count個すべてについて1〜2を行い、常にcount個の風船が同時に稼働する状態にする(ヒット時は下記の補充規則で即座に1個追加する)。

**ヒット時(onTouch, target ≠ null)**:
1. `target.data.ctrl.stop()`で移動を止め、`ctx.score += 1`。
2. `ctx.feedback.success(target.el)`を呼び、`ctx.removeTarget(target.el)`後に`el`を除去する。
3. 出現規則1〜2に従って新しい風船を1個、即座に補充する(常にcount個を維持する)。

**空振りタッチ時(onTouch, target === null)**: `ctx.data.misses += 1`のみ行う(視覚・聴覚フィードバックは一切出さない)。

**タイマー表示**: durationは常に時間制(30s/60s/90s)のため、coreのTimerBarを表示する。回数制モードは持たない。

### 4-2. 先生モードでの表示(差分)

extras(命中率)の直近履歴が表示される。本アプリ固有の追加表示はなし。

### 4-3. 一時停止・設定変更・中断

共通仕様のまま(設定変更ありで閉じたらリスタート、中断は記録なし)。一時停止中は各風船の`VT.Motion.move`のctrlがcoreのシーンpauseと自動連動して静止し、再開(無変更時)で同じ軌道のまま動きを再開する。ターゲット・風船要素はcoreのシーン破棄で消えるため`onAbort`は省略可。

## 5. 受け入れ条件(Definition of Done)

**共通DoD**: ai-notes.md §9 の8項目すべて。

**個別DoD**:
1. ★2相当の設定(size=xl, speed=slow, motion=straight, count=1, duration=30s)で、風船以外へどれだけ連打しても画面上・音に一切の負のフィードバックが生じない(命中率算出のための内部カウントのみ行われる)。
2. count=3のとき、常に3個の風船が独立した軌道で動き続け、稼働数が3を下回ったまま放置されない。
3. speed=fastかつsize=sでも、風船がステージ範囲内に収まって動き続け、画面外へ消えたり停止したりしない(iPad実機で確認)。
4. motion=straightの風船はまっすぐ往復する軌道を、motion=wanderの風船はふらふらとした軌道を描き、見た目で区別できる。
5. 先生パネルを開いて全風船の移動が止まり、無変更で閉じると同じ軌道のまま動きを再開する。
6. duration満了時、稼働中の風船の有無にかかわらず結果画面へ正しく遷移し、hits・hitRateが妥当な値(タッチ0回の場合はhitRate=0)で記録される。
7. 結果画面の前回比が、digestまたは日付が異なる直前記録に対して表示されない。

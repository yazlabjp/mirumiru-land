## 1. 概要とねらい(3行以内)

画面に1つずつ(設定で最大3つ)現れる絵カードを、見つけてタッチするだけのアプリ。視覚探索と目と手の協応の最基礎を担い、シリーズ全体の玄関・タッチ操作の練習・実態観察の3役を果たす。触れば必ず成功する★1設定から、小さく速い★3設定まで、設定のみで難易度が動く。

## 2. 用語と対象物の定義

- **ターゲット**: VT.Stimの絵カード1枚。タッチ対象。常に `ctx.addTarget()` で登録する。
- **ヒット**: ターゲットへの有効タッチ(コアのデバウンス通過後にonTouchへ届いたもの)。
- **ミスタッチ**: ターゲット外へのタッチ。**完全に無視する**(視覚・聴覚フィードバックなし。本バージョンでは内部カウントもしない)。
- **RT(反応時間)**: そのターゲットの出現時刻(`performance.now()`)から、ヒットの `target.timeStamp` までのms。ターゲットごとに計測する(出現時刻は `ctx.addTarget(el, {shownAt})` のdataに保持)。

## 3. 設定スキーマ(完全版・このまま実装する)

```javascript
settings: {
  size: { type: "choice", label: "大きさ", default: "xl",
    options: [ {value:"xl",label:"特大"}, {value:"l",label:"大"},
               {value:"m",label:"中"},  {value:"s",label:"小"} ] },
  area: { type: "choice", label: "出現範囲", default: "full",
    options: [ {value:"center",label:"中央"}, {value:"full",label:"全画面"},
               {value:"left",label:"左半分"}, {value:"right",label:"右半分"},
               {value:"bottom",label:"下半分"} ] },
  count: { type: "range", label: "同時に出る数", min: 1, max: 3, step: 1, default: 1 },
  pace: { type: "choice", label: "つぎが出るまで", default: "normal",
    options: [ {value:"fast",label:"すぐ"}, {value:"normal",label:"ふつう"},
               {value:"slow",label:"ゆっくり"} ] },
  duration: { type: "choice", label: "時間", default: "60s",
    options: [ {value:"30s",label:"30秒"}, {value:"60s",label:"1分"},
               {value:"120s",label:"2分"}, {value:"n10",label:"10回タッチ"} ] },
  theme: { type: "choice", label: "絵柄", default: "animals", difficulty: false,
    options: [ {value:"animals",label:"どうぶつ"}, {value:"fruits",label:"くだもの"},
               {value:"vehicles",label:"のりもの"} ] }
}
```

**数値対応表(規範)**:

| 設定 | value → 実値 |
|---|---|
| size | xl→180px / l→140px / m→110px / s→80px(カードの一辺) |
| pace | fast→0ms / normal→400ms / slow→900ms(ヒット後、補充ターゲット出現までの遅延) |

- digest対象は size/area/count/pace/duration(定義順)。例: `"xl/full/1/normal/60s"`。themeは `difficulty:false` で除外。
- 共通設定(sound/reduceMotion/mirror/contrast)はcoreが自動付与する(vt-api §2-2)。**mirrorの本アプリでの意味**: `mirror:true` のとき、areaの `left` と `right` を入れ替えて解釈する(それ以外のareaでは無効果)。

## 4. 記録スキーマ

```javascript
record: {
  primary: { key: "hits",  label: "タッチできた かず", betterIs: "higher" },
  extras:  [ { key: "avgRt", label: "平均反応時間", unit: "ms" } ]
}
```

- avgRt = 全ヒットRTの単純平均(四捨五入で整数ms)。ヒット0件時は0。
- 既知の制約(仕様として許容): duration="n10" ではhitsが常に10になり前回比が平坦になる。n10の観察対象はavgRt(先生モード)とする。

## 5. 画面と挙動(共通3画面フローとの差分のみ)

### 5-1. タイトル画面

共通仕様のまま(差分なし)。スタートボタンのpointerdownでAudioContext生成(core側・vt-api §6)。

### 5-2. プレイ画面

**開始(onStart)**: `ctx.data` に `{ rts: [], targets: [] }` を用意し、`count` 個のターゲットを出現規則(下記)で配置する。

**出現規則(規範)**:
1. カードは `VT.Stim.random(theme)` で選ぶ。**画面上に同時に存在するカードは絵柄が重複しないよう**、現存カード名を `exclude` に渡す。
2. 位置は `VT.Rand.pos({size, area, avoid})` で得る。`avoid` には直前にヒットしたターゲットの位置(初回はなし)を渡す。
3. count>1 のときの重なり回避はアプリ側で行う: 得た位置が既存ターゲットと**中心間距離 < size** なら再抽選し、最大20回で妥協して採用する(無限ループ禁止)。
4. **誘導ヒント(原則2)**: 各プレイの最初の1個だけ、scale 1.0↔1.08・周期1200msのゆっくりした明滅を、最初のヒットまで続ける。`reduceMotion:true` のときは明滅せず、代わりに外周に太い白フチ(8px)を付ける。

**ヒット時(onTouch, target ≠ null)**:
1. RTを `ctx.data.rts` に記録し、`ctx.score += 1`。
2. `ctx.feedback.success(target.el)` を呼ぶ(演出スタイルは"auto"のまま)。
3. 当該ターゲットを `ctx.removeTarget(el)` し、演出と干渉しないよう即座に `el` を除去(除去アニメはFeedbackFXに任せる)。
4. `pace` の遅延後に補充ターゲットを1個出現させ、常に `count` 個を維持する。遅延の計時はonTick内のdt積算で行う(`setTimeout` 禁止・ai-notes §1)。
5. duration="n10" のとき、`ctx.score` が10に達したら補充せず `ctx.end()`。

**ミスタッチ時**: 何もしない(§2)。`stageTouch` は使わない(既定false)。

**タイマー表示**: 時間制(30s/60s/120s)はcoreのTimerBar。n10はTimerBar非表示(§7の前提参照)。

### 5-3. できた!画面

共通仕様のまま。primaryの大表示は「タッチできた かず」。前回比(同digest・同日)・ベスト演出はcore任せ。

### 5-4. 一時停止・設定変更・中断

すべて共通仕様のまま(設定変更ありで閉じたらリスタート、中断は記録なし)。アプリ固有の後始末はなし(ターゲットはcoreのシーン破棄で消えるため `onAbort` は省略可)。

## 6. 先生モードでの表示

共通仕様のまま: extras(平均反応時間)の直近履歴が表示される。本アプリ固有の追加表示はなし(領域別ヒートマップは§8で対象外とした)。

## 7. coreへの前提(API依存事項)

本アプリが使用するAPI: `VT.createApp / ctx(settings, data, score, stage, rect, dt, addTarget, removeTarget, end, feedback) / VT.Stim.random / VT.Rand.pos / VT.Rand.int`。すべてvt-api.md記載済み。

**本specで確定するvt-api §13の未確定事項(実装時にvt-api.mdへ転記すること)**: coreはsettingsキー名 `duration` を予約的に解釈する。

| duration value | coreの挙動 |
|---|---|
| `"30s"` / `"60s"` / `"120s"` | 時間制。TimerBar表示。満了で自動的にonFinish |
| `"n10"` | 自動満了なし・TimerBar非表示。終了はアプリの `ctx.end()` に委ねる |

## 8. 対象外(本バージョンでやらないこと)

- 画面領域別ヒートマップ(設計書C-01の評価方法にあるが、先生パネルへの独自ビュー追加APIが未整備のため**v2へ延期**。必要API: 先生モード内カスタム表示領域)。
- ミスタッチの内部カウント・記録。
- 動くターゲット(C-03の領分)/妨害キャラ(D-05の領分)。
- 2人モード(E-06の領分)。

## 9. 受け入れ条件(Definition of Done)

**共通DoD**: ai-notes.md §9 の8項目すべて。

**個別DoD**:
1. ★1相当の設定(size=xl, area=center, count=1, pace=slow, duration=30s)で、ターゲット以外をどれだけ連打しても画面上・音・記録に一切の負のフィードバックが生じない。
2. count=3 でターゲット同士が重ならずに出現する(20回試行の妥協採用時を除き、視認上の重なりがない)。
3. 各プレイの最初のターゲットに誘導ヒント(明滅または白フチ)が付き、最初のヒットで消える。
4. duration=n10 で10ヒット目の直後に結果画面へ遷移し、hits=10・avgRtが妥当な値で記録される。
5. mirror=ON かつ area=left のとき、ターゲットが右半分に出現する。
6. 結果画面の前回比が、digestまたは日付が異なる直前記録に対して表示されない。
7. iPad実機で、ヒット時の演出・補充のあいだも入力が取りこぼされない(連続タッチのテンポ良い進行)。

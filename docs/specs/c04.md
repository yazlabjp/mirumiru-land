## 1. 概要とねらい

上から落ちてくる果物を、画面下のカゴを指で左右にドラッグして受け止める、動体視力(落下してくる対象を捉える目の動き)と落下予測(到達点をあらかじめ見積もる力)のアプリ。落下速度・カゴの幅・同時に落ちる数を設定で調整でき、★2のゆっくり大きいカゴの設定から★4の速く狭いカゴの設定まで難易度が動く。「取ってはいけない物の混入(抑制課題化)」は設計書上の将来発展案であり、本バージョンでは実装しない。

## 2. 設定スキーマ(完全版・このまま実装する)

```javascript
settings: {
  fallSpeed: { type: "choice", label: "おちる はやさ", default: "slow",
    options: [ {value:"slow",label:"ゆっくり"}, {value:"normal",label:"ふつう"},
               {value:"fast",label:"はやい"} ] },
  basketWidth: { type: "choice", label: "カゴの はば", default: "xl",
    options: [ {value:"xl",label:"とても ひろい"}, {value:"l",label:"ひろい"},
               {value:"m",label:"ふつう"}, {value:"s",label:"せまい"} ] },
  count: { type: "range", label: "どうじに おちる かず", min: 1, max: 3, step: 1, default: 1 },
  duration: { type: "choice", label: "時間", default: "60s",
    options: [ {value:"30s",label:"30秒"}, {value:"60s",label:"1分"},
               {value:"90s",label:"1分30秒"} ] }
}
```

**数値対応表(規範)**:

| 設定 | value → 実値 |
|---|---|
| fallSpeed | slow→120px/秒 / normal→200px/秒 / fast→320px/秒(果物が真下へ落下する速度) |
| basketWidth | xl→320px / l→260px / m→200px / s→150px(カゴの当たり判定の幅=見た目の幅。`ctx.rect()`実寸を基準に中央付近へ初期配置) |

- 果物のサイズ(直径120px固定)・カゴの高さ(70px固定)は設計書のC-04行に難易度設定として挙げられていないため、設定化せずapp.js内の固定値とする。
- digest対象は fallSpeed/basketWidth/count/duration(定義順)。例: `"slow/xl/1/60s"`。difficulty:falseの設定はなし(絵柄固定のためtheme相当の設定を持たない。果物の絵柄はC-01と同じ`VT.Stim.random("fruits")`のテーマを流用する)。
- 共通設定(sound/reduceMotion/mirror/contrast)はcoreが自動付与する(vt-api §2-2)。mirrorは左右対称のドラッグ操作に意味を持たないため無効果(C-03と同様の扱い)。reduceMotion:trueでも果物の落下自体(課題に必須の動き)は影響を受けず、キャッチ演出のみ簡略化される(vt-api §6)。

## 3. 記録スキーマ

```javascript
record: {
  primary: { key: "catches",   label: "キャッチできた かず", betterIs: "higher" },
  extras:  [ { key: "catchRate", label: "キャッチ率", unit: "%" } ]
}
```

- catchRate = キャッチ数 ÷(キャッチ数 + 取りこぼし数)× 100(四捨五入で整数%)。1個も処理されないまま終了した場合は0。
- 取りこぼし(カゴに当たらずステージ下端を通過した果物)はcommon規則により視覚・聴覚とも一切の反応を出さない(§4参照)。キャッチ率算出のためだけに内部カウントする観察用の値。

## 4. 画面と挙動(共通3画面フローとの差分のみ)

### 4-1. プレイ画面

**実装方式についての前提**: 本アプリはC-01〜C-03と異なり、`ctx.addTarget`/`onTouch`によるタップ判定を使用しない(果物は直接タッチする対象ではなく、カゴで受け止める対象のため)。カゴは`VT.Input.drag`で直接ドラッグ制御し、果物は`VT.Motion.move`(往復パスのため落下向きではない)を使わず、`onTick`内の`ctx.dt`積算で自前にy座標を進める。カゴの見た目は`VT.Stim.shape`に該当形状がないため、app.js内で直接SVG要素を組み立てる(vt-api.mdに存在しないVT関数を呼び出すわけではなく、装飾要素の静的マークアップを自作するのみのため、ai-notes §0-4の規約には抵触しない)。

**開始(onStart)**: `ctx.data`に`{ misses: 0, basketX: (ctx.rect().w - basketWidthPx) / 2, fruits: [] }`を用意する。カゴ要素を`ctx.stage`直下の子として1個生成し、ステージ下端付近(`y = ctx.rect().h - 110`固定・ドラッグは左右のみ)に配置、`VT.Input.drag(basketEl, {onMove}, {clampToStage:true})`をアタッチする。`onMove`では受け取った`x`を基準にカゴ中心を追従させ、`ctx.data.basketX`(カゴ左端)を更新してスタイルに反映する(`clampToStage:true`によりステージ外へは出ない)。続けて`count`個の果物を下記の出現規則で生成する。

**出現規則(規範)**:
1. `VT.Stim.random("fruits", {})`で果物を1個生成し、ステージ上端(`y = -120`)の外側から出現させる。x位置は`VT.Rand.int(24, ctx.rect().w - 24 - 120)`で決め、既存の落下中の果物と中心間距離が120px未満なら再抽選(最大20回で妥協して採用。無限ループ禁止)。
2. 生成した果物要素へ`{x, y, speed}`(speedは`fallSpeed`の実値)を紐づけて`ctx.data.fruits`配列に保持する(`ctx.addTarget`は使わない)。
3. count個すべてについて1〜2を行い、常にcount個の果物が同時に落下している状態を保つ(キャッチ・取りこぼしいずれでも即座に1個補充する)。

**onTickでの落下・判定(規範)**:
1. `ctx.data.fruits`の各果物について`y += speed * ctx.dt / 1000`し、要素のスタイルへ反映する。
2. **キャッチ判定**: 果物の中心y座標がカゴ上端(`ctx.rect().h - 110`)以上に達し、かつ果物の中心xが`[ctx.data.basketX - 60, ctx.data.basketX + basketWidthPx + 60]`の範囲内であれば、キャッチ成立とする。`ctx.score += 1`、`ctx.feedback.success(el)`を呼び、果物を`ctx.data.fruits`から取り除いて`el`を除去し、出現規則1〜2に従って新しい果物を1個補充する。
3. **取りこぼし判定**: キャッチ条件を満たさないまま果物の中心yが`ctx.rect().h`を超えたら、`ctx.data.misses += 1`のみ行い(視覚・聴覚フィードバックは一切出さない)、果物を`ctx.data.fruits`から取り除いて`el`を除去し、出現規則1〜2に従って新しい果物を1個補充する。

**ミスタッチ時**: カゴ以外の場所へのタッチは完全に無視する(ドラッグ操作用のポインタ入力以外は判定を持たない)。`stageTouch`は使わない(既定false)。

**タイマー表示**: durationは常に時間制(30s/60s/90s)のため、coreのTimerBarを表示する。回数制モードは持たない。

### 4-2. 先生モードでの表示(差分)

extras(キャッチ率)の直近履歴が表示される。本アプリ固有の追加表示はなし。

### 4-3. 一時停止・設定変更・中断

共通仕様のまま(設定変更ありで閉じたらリスタート、中断は記録なし)。一時停止中はcoreが`onTick`を呼ばないため、果物の落下は自動的に静止する(`VT.Motion.move`のctrl連動ではなく、dt積算がonTick停止と同時に止まる形で実現)。先生パネル表示中はパネルがステージ上をカバーするため、カゴのドラッグ操作も実質的に受け付けなくなる。無変更で閉じた場合は「3・2・1」表示後、同じ位置・状態のまま再開する。カゴ・果物要素はcoreのシーン破棄で消えるため`onAbort`は省略可。

## 5. 受け入れ条件(Definition of Done)

**共通DoD**: ai-notes.md §9 の8項目すべて。

**個別DoD**:
1. ★2相当の設定(fallSpeed=slow, basketWidth=xl, count=1, duration=30s)で、果物を取りこぼしても画面上・音に一切の負のフィードバックが生じない(キャッチ率算出のための内部カウントのみ行われる)。
2. count=3のとき、常に3個の果物が独立した位置・タイミングで落下し続け、稼働数が3を下回ったまま放置されない。
3. fallSpeed=fastかつbasketWidth=s(せまい)でも、カゴはステージ内に収まって左右にドラッグでき、ステージ外へはみ出さない(iPad実機で確認)。
4. カゴの当たり判定幅が数値対応表のpx値(見た目の幅)と一致し、見た目のカゴの外側でキャッチが成立しない。
5. 先生パネルを開くとカゴのドラッグ操作と全果物の落下が停止し、無変更で閉じると同じ位置・状態のまま再開する。
6. duration満了時、落下中の果物の有無にかかわらず結果画面へ正しく遷移し、catches・catchRateが妥当な値(キャッチ0回の場合catchRate=0)で記録される。
7. 結果画面の前回比が、digestまたは日付が異なる直前記録に対して表示されない。

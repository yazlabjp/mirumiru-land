## 1. 概要とねらい

穴から顔を出すもぐらを見つけてタッチする、出現位置への素早い到達運動と予測反応のアプリ。もぐらが出ている時間を「出しっぱなし」〜「すぐ引っ込む」まで調整でき、★2の緩い設定から★4の速い設定まで難易度が動く。設定でNGキャラを混ぜると簡単な弁別課題も加わる(D-05への前段接続)。

## 2. 設定スキーマ(完全版・このまま実装する)

```javascript
settings: {
  holes: { type: "choice", label: "あなの かず", default: "6",
    options: [ {value:"4",label:"4つ"}, {value:"6",label:"6つ"}, {value:"9",label:"9つ"} ] },
  popTime: { type: "choice", label: "でている じかん", default: "normal",
    options: [ {value:"long",label:"ながく"}, {value:"normal",label:"ふつう"},
               {value:"fast",label:"みじかく"} ] },
  count: { type: "range", label: "どうじに出る かず", min: 1, max: 3, step: 1, default: 1 },
  ngMole: { type: "toggle", label: "ちがうこも まじる", default: false },
  duration: { type: "choice", label: "時間", default: "60s",
    options: [ {value:"30s",label:"30秒"}, {value:"60s",label:"1分"},
               {value:"90s",label:"1分30秒"} ] }
}
```

**数値対応表(規範)**:

| 設定 | value → 実値 |
|---|---|
| holes | 4→2×2グリッド / 6→3×2グリッド(横3・縦2) / 9→3×3グリッド(いずれも`ctx.rect()`実寸から等間隔算出) |
| popTime | long→2000ms / normal→1200ms / fast→700ms(穴から出てから、タッチされずに自動で引っ込むまでの時間) |
| ngMole | true時、もぐら出現のたびに約10%の確率でNGキャラに差し替え |

- digest対象は holes/popTime/count/ngMole/duration(定義順)。例: `"6/normal/1/false/60s"`。difficulty:falseの設定はなし(絵柄固定のためtheme相当の設定を持たない)。
- 共通設定(sound/reduceMotion/mirror/contrast)はcoreが自動付与する(vt-api §2-2)。mirrorはholesグリッドの列順を左右反転する形で解釈する。

## 3. 記録スキーマ

```javascript
record: {
  primary: { key: "hits",  label: "たたけた かず", betterIs: "higher" },
  extras:  [ { key: "avgRt", label: "平均到達時間", unit: "ms" },
             { key: "ngTouched", label: "ちがうこを タッチした かず", unit: "回" } ]
}
```

- avgRt = 通常もぐらへの全ヒットRT(もぐらが穴からせり上がりきった時刻→タッチの`target.timeStamp`)の単純平均(四捨五入で整数ms)。ヒット0件時は0。
- ngTouched = NGキャラをタッチしてしまった回数。得点には影響しない観察用の値。ngMole=falseで遊んだ回は常に0のまま記録される(既知の仕様)。

## 4. 画面と挙動(共通3画面フローとの差分のみ)

### 4-1. プレイ画面

**開始(onStart)**: `ctx.data`に`{ rts:[], ngTouched:0, holePositions:[], activeCount:0 }`を用意する。`holes`設定に応じたグリッド(4→2×2/6→3×2/9→3×3)の穴座標を`ctx.rect()`の実寸から等間隔で算出し、各穴を非タッチ対象の背景要素として`ctx.stage`に静的配置する(`addTarget`しない・穴自体は常時表示のまま消えない)。続けて空いている穴から`count`個を選び、下記の出現規則でもぐらを出す。

**出現規則(規範)**:
1. 空いている穴を`VT.Rand.pick`で1つ選ぶ(直前にヒット/引っ込みが起きた穴は次の抽選で除外し、同じ穴の連続出現を避ける)。
2. `ngMole:true`のとき、出現のたびに約10%の確率でNGキャラ(もぐらと明確に異なる見た目・色)を出す。それ以外は通常のもぐら。
3. `VT.Motion.tween`で穴からせり上がる演出(y方向・200ms程度)を行った後、`ctx.addTarget(el, {shownAt, isNG})`でタッチ対象として登録する。
4. `popTime`(long/normal/fast→2000/1200/700ms)が経過してもタッチされなければ、静かに引っ込むtweenの後`ctx.removeTarget(el)`し、その穴を空きに戻す(失敗演出・失敗音は一切出さない)。
5. ヒット・タイムアウトいずれで穴が空いても、常に稼働中のもぐら数が`count`を保つよう、短い間隔を空けて別の空き穴に次の出現を行う。間隔の計時は`onTick`内の`dt`積算で行う(`setTimeout`禁止・ai-notes §1)。

**ヒット時(onTouch, target ≠ null)**:
1. `target.data.isNG`がfalseのとき: RTを`ctx.data.rts`に記録し`ctx.score += 1`、`ctx.feedback.success(target.el)`を呼んで即座に穴を空きに戻す。
2. `target.data.isNG`がtrueのとき: `ctx.data.ngTouched += 1`。`ctx.feedback.soft(target.el)`(静かな揺れのみ・音なし)を呼び、得点は変えずに穴を空きに戻す。
3. いずれの場合も4-1の出現ループへ戻り、次のもぐら(またはNGキャラ)を補充する。

**ミスタッチ時**: 穴以外・まだ何も出ていない穴へのタッチは完全に無視する(視覚・聴覚フィードバックなし・内部カウントもしない)。`stageTouch`は使わない(既定false)。

**タイマー表示**: durationは常に時間制(30s/60s/90s)のため、coreのTimerBarを表示する。回数制モードは持たない。

### 4-2. 先生モードでの表示(差分)

extras(平均到達時間・NGタッチ回数)の直近履歴が表示される。本アプリ固有の追加表示はなし。

### 4-3. 一時停止・設定変更・中断

共通仕様のまま(設定変更ありで閉じたらリスタート、中断は記録なし)。設定変更で再スタートした場合、穴のグリッドは新しい`holes`値で再計算される。ターゲット・穴要素はcoreのシーン破棄で消えるため`onAbort`は省略可。

## 5. 受け入れ条件(Definition of Done)

**共通DoD**: ai-notes.md §9 の8項目すべて。

**個別DoD**:
1. ★1相当の設定(holes=4, popTime=long, count=1, ngMole=false, duration=30s)で、穴以外や何も出ていない穴をどれだけ連打しても、画面上・音・記録に一切の負のフィードバックが生じない。
2. holes=9のとき、9つの穴が重ならず画面内に収まって等間隔に配置される(iPad実機の縦横どちらの実寸でも)。
3. count=3のとき、常に3体のもぐら(またはNGキャラ)が独立したタイミングで出没を繰り返し、稼働数が3を下回ったまま放置されない。
4. popTime=fastでも、出現→タッチ→引っ込みの演出が破綻せず連続する(iPad実機で連続タッチの取りこぼしがない)。
5. ngMole=ONのとき、NGキャラをタッチしても得点・成功演出・効果音のいずれも発生せず、静かな反応(soft)のみが起こる。ngMole=OFFのときはNGキャラが一切出現しない。
6. duration満了時、アクティブなもぐら・NGキャラの有無に関わらず結果画面へ正しく遷移し、hits・avgRt・ngTouchedが妥当な値で記録される。
7. 結果画面の前回比が、digestまたは日付が異なる直前記録に対して表示されない。

## 1. プロダクトの前提(これだけは常に念頭に)

特別支援学校の**自立活動(個別課題)**で使うビジョントレーニングWebアプリ群。利用者は知的障害・発達障害のある小学部〜高等部の児童生徒(実態は幅広く混在)と、隣にいる教員。環境は**学校iPadのSafari・横持ち・タッチのみ**。GitHub Pagesで公開する静的サイト(サーバーなし・ビルドなし・依存ゼロ)で、将来オフライン動作(PWA)にする。記録は端末内のみ・個人情報ゼロ。

## 2. 7つの設計原則(実装判断に迷ったらここに戻る)

1. **タッチ2回で始まる** — カタログでアプリを選ぶ→スタート。メニュー階層を追加しない。
2. **読めなくても遊べる** — 文字と音声はすべて補助。画面を見れば何をするか分かる誘導(最初の1個が光る等)を必ず用意する。
3. **失敗を鳴らさない** — 負のフィードバック(ブザー・バツ・減点表示・暗転・ミス数の表示)は全面禁止。正解だけが華やかに反応し、不正解は無反応か静かな揺れのみ。
4. **すべてのアプリに★1の入り口** — 最も易しい設定にすれば「触れば必ず成功する」状態まで下がること。難易度はアプリを分けずに設定で動かす。
5. **先生の30秒調整** — 難易度に関わる要素はハードコードせず設定スキーマに出す。設定は自動保存。
6. **個人情報ゼロ** — 名前・写真・アカウントを扱わない。リポジトリに写真・実名・学校固有情報を置かない。写真機能は端末内File API読込のみ。
7. **1本1機能** — specにない機能を足さない。足したい改善は実装せず提案として報告する。

## 3. 画面とUIの共通規則

- **画面フロー**: タイトル →(スタート)→ プレイ →(終了)→ できた! →「もういっかい」/「おわる」。全アプリ共通・coreが提供。アプリが独自画面を作らない。
- **タッチターゲット**: 児童生徒が押す要素は最小 **72×72px**(≒15mm)、課題の主対象は原則 **120px以上**、要素間隔16px以上。「小さい」は設定による上級化としてのみ許す。
- **文字**: 児童生徒向けはひらがな+分かち書き(例「もう いっかい」)・最小24px。先生パネル内は漢字かなまじり可。フォントはcore同梱のBIZ UDPGothic(自動適用)。
- **時間表示**: 数字でなく減っていくバー(core提供)。残りわずかの点滅・警告音は禁止。
- **ジェスチャ**: 児童生徒操作は**タップとドラッグの2つだけ**。スワイプ・ピンチ・長押し・ダブルタップ禁止(先生モードの3秒長押しはcore内部のみ)。
- **姿勢**: 横持ち前提。縦持ち時はcoreのOrientationGuardが自動対応(アプリ側の実装不要)。
- **アクセシビリティ**: 点滅は3Hz以下。音なしで全課題が成立すること。共通設定 `reduceMotion`(演出簡略化)と `mirror`(左右反転。アプリごとの解釈はspecに定義)に対応する。
- **先生モード**: coreが設定スキーマから自動生成。アプリ側は「難易度に関わる値をすべてスキーマに出す」ことだけが責務。

## 4. カラーシステム(themes.cssのCSS変数を必ず使う)

色の直書き(hexベタ書き)をせず、以下の変数を参照する。

| 用途 | 変数 | 値 | 備考 |
|---|---|---|---|
| 系統色 | `--cat-a`〜`--cat-f` | A:#4FB6E0 B:#56B87A C:#F59A3E D:#9B7BC8 E:#E88AA8 F:#8A939E | アイコン・タイトル帯用。A目のうんどう/B見てわかる/C見てうごく/Dおぼえる・きづく/Eつかう/Fはかる |
| 背景 | `--bg` | 既定#FAF7F0(生成り) | 共通設定contrastで白/黒に切替(core処理) |
| 文字 | `--ink` | #3A3A3A | 真っ黒を使わない |
| 刺激色 | `--stim-red / -blue / -yellow / -green` … | #E64545 / #3B76D6 / #F5C63C / #43A356 | 課題の刺激専用の高彩度セット |

**二重符号化の規則**: 正解/不正解・Go/NoGoなど**意味の区別を色だけで伝えない**(色+形、色+位置、色+動きを併用)。色弁別そのものが課題の場合(B-01等)のみ例外で、specに明記する。

## 5. アイコン(各アプリのicon.svg)

系統色ベタ塗りの角丸正方形(角丸率22%)+**白1色・太線のモチーフ1つ**(線幅は512px基準で32px相当)。文字・写真・モチーフ2つ以上・グラデーション禁止。viewBoxは0 0 512 512。

## 6. リポジトリ規約

```
mirumiru-land/
├── .nojekyll  LICENSE(MIT)  README.md  index.html  apps.json
├── core/v1/          ← css/ js/ assets/(fonts, stim) test.html
├── apps/<id>/        ← index.html app.js app.css icon.svg(meta.jsonは無い)
├── _template/        ← 新作の雛形
└── docs/             ← developer-guide.md vt-api.md ai-notes.md specs/ checklist.md ほか
```

- **id命名**: `系統コード小文字+2桁連番-英語スラッグ`(例 `c01-target-touch`)。フォルダ名=meta.id=apps.jsonのidを完全一致させる。
- **アプリ情報の唯一の情報源はapps.json**。アプリ追加=フォルダを置く+apps.jsonに1項目(specの§10に貼り付け用エントリがある)。
- **coreは後方互換のみ**: core/v1内は関数追加のみ可・挙動変更不可。破壊的変更はcore/v2を新設(旧アプリは触らない)。
- パスはすべて相対参照(アプリからcoreへは `../../core/v1/...`)。

## 7. 新作実装の手順(1本道)

1. `_template` をコピーし、フォルダ名をidに変更(index.htmlの雛形は変更しない)。
2. 該当spec(docs/specs/<id>.md)を受領・熟読(specが無い作業は開始しない)。
3. app.jsを実装(vt-api.mdにあるAPIのみ使用。無い機能は追加提案として報告)。
4. icon.svgを§5の規則で作成。
5. apps.jsonにspec §10のエントリを追加(status: beta)。
6. specの受け入れ条件(共通DoD=ai-notes §9+個別DoD)を自己検証し、結果を○×で報告。
7. core/v1/test.htmlが全項目○であることを確認(coreに触れた場合は必須)。

## 8. 難易度と設定の考え方(実装者が守る精神)

対象の児童生徒は「画面の変化に気づく段階」から「速さと正確さに挑戦する段階」まで4ステージにわたる。specの設定スキーマはこの幅を1本のアプリでカバーするために設計されているので、**既定値=最も易しい側**(原則4)を崩さないこと。難易度に影響する設定はdigest(記録の比較単位)に入るため、リリース後にvalueや選択肢の意味を変えない(vt-api §2)。

## 9. 本ガイドの更新

デザイン規則・リポジトリ規約の変更は、設計書の改訂と同時に本ガイドへ反映する(版数と日付を更新)。実装AIは常に最新版の4点パッケージを正とする。
